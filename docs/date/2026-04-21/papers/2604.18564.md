# MultiWorld: Scalable Multi-Agent Multi-View Video World Models

**【多智能体多视角视频世界建模】**

**arXiv**: https://arxiv.org/abs/2604.18564

## 简要摘要

*计算机视觉;视频生成;多智能体世界模型*

现有视频世界模型难以处理多智能体交互和多视角一致性问题。
提出MultiWorld框架，通过多智能体控制模块和全局状态编码器，实现可控多智能体与多视角一致的高效视频生成。

## 深度解析

这篇论文的核心内容可以概括如下：

### (1) 任务是什么？
- **任务名称**：多智能体多视角视频世界模型。
- **任务输入**：
    - **历史视频帧**：过去几帧多个摄像头（视角）拍摄的视频。
    - **当前动作**：多个智能体在同一时刻各自要执行的动作指令。
- **任务输出**：
    - **未来视频帧**：预测出下一时刻（或连续多帧）所有摄像头视角下的视频。
- **任务功能**：
    - 模拟一个由多个智能体共同行动的环境动态。
    - 例如，预测两个玩家在游戏中合作行动时，各自屏幕（视角）上会看到什么；或者多个机器人协同搬运物体时，不同位置的摄像头会拍到什么画面。

### (2) 想解决什么问题？
现有的“视频世界模型”大多只能模拟**单个智能体**的行动，并生成**单一视角**的未来画面。但在真实世界中（如多玩家游戏、多机器人协作），往往是：
1.  **多个智能体同时行动并相互影响**。
2.  每个智能体有自己的观察视角（摄像头）。
3.  智能体和视角的数量可能变化。

因此，论文想解决三个核心挑战：
- **多智能体可控性**：如何让模型准确理解“哪个动作属于哪个智能体”，并让每个智能体都按指令行动？
- **多视角一致性**：不同视角拍的是同一个共享的3D环境，如何保证生成的不同视角画面在物理上是连贯、不矛盾的？（例如，一个视角里机器人A拿着球，另一个视角里不能显示球凭空消失）。
- **框架可扩展性**：如何设计一个模型，能灵活处理不同数量的智能体和视角，而不是固定死只能处理两个智能体或三个摄像头？

### (3) 核心思路是什么？
论文提出了一个名为 **MultiWorld** 的统一框架，核心是设计了两个关键模块：

1.  **多智能体条件模块**：
    - 给每个智能体分配一个独特的“身份嵌入”，就像给每个人一个ID卡。这样模型就能区分不同智能体的动作。
    - 引入“自适应动作加权”，动态关注那些正在执行重要动作的智能体，忽略静止不动的，让模型学习更高效。

2.  **全局状态编码器**：
    - 把所有不同视角的观察（视频帧）整合起来，编码成一个统一的、对3D空间有感知的“全局环境状态”。可以把它想象成在电脑里构建了一个当前时刻的3D场景快照。
    - 生成每个视角的未来画面时，都基于这个**同一个全局状态**来生成，这样就自然保证了不同视角画面的一致性。

通过这两个模块，模型实现了**智能体可控**和**视角一致**，并且因为身份嵌入和全局状态的设计是灵活的，所以能**扩展到任意数量的智能体和视角**。

### (4) 结果怎么样？
- **数据集**：论文构建了两个新数据集进行测试：
    1.  **《双人成行》游戏数据集**：双智能体，各有视角，动作相互依赖。
    2.  **多机器人操作数据集**：智能体和摄像头数量可变。
- **评估指标**：从**视频质量**、**动作跟随准确性**、**多视角一致性**（如重投影误差）等方面进行评估。
- **结论**：MultiWorld 在视频逼真度、智能体对动作指令的遵从度、以及不同视角间的一致性上，都显著超过了其他基线方法。

### (5) 模型结构与整体流程
论文没有明确说基于某个特定“基模”，但从描述看，主体应该是一个**以Transformer为核心的动作条件视频生成模型**。整体流程可以参照方法部分（Sec. 3）描述来理解：

1.  **输入准备**：
    - **历史帧**：假设有 `N` 个视角，每个视角提供过去 `T` 帧画面。所有画面被编码成特征。
    - **当前动作**：`M` 个智能体在当前时刻的动作指令，每个动作被编码成向量。

2.  **多智能体条件处理**：
    - 动作向量输入 **多智能体条件模块**。
    - **步骤1：身份嵌入**：为每个智能体的动作向量加上一个独特的身份编码（Agent Identity Embedding），这样模型就能区分“动作1是机器人A的，动作2是机器人B的”。
    - **步骤2：自适应加权**：计算每个智能体动作的“重要性权重”，活跃智能体的权重高，静止的权重低。然后将加权后的所有智能体动作信息融合成一个**统一的多智能体条件信号**。

3.  **构建全局环境理解**：
    - 所有 `N` 个视角的历史帧特征，输入 **全局状态编码器**。
    - 该编码器（可能利用3D感知网络，如基于Transformer的架构）将这些多视角2D特征融合、推理，输出一个紧凑的 **3D感知全局状态特征**。这个特征代表了当前时刻整个共享环境的统一状态。

4.  **生成未来帧**：
    - **条件组合**：将 **统一的多智能体条件信号** 和 **3D感知全局状态特征** 结合起来，作为视频生成模型的“条件输入”。
    - **并行生成**：视频生成模型（如扩散模型或流匹配模型）以这个组合条件为指导，**并行地** 为每一个指定的目标视角生成未来的一帧（或多帧）画面。因为所有视角的生成都基于同一个全局状态和同一组动作条件，所以结果自然协调一致。

5.  **输出**：
    - 得到 `N` 个视角在下一时刻（或未来序列）的预测视频帧。

**总结流程图**：
`多视角历史帧 + 多智能体当前动作`
→ `[全局状态编码器] + [多智能体条件模块]`
→ `统一的3D全局状态 + 区分智能体的动作条件`
→ `视频生成模型（条件输入）`
→ `并行生成 → 多视角一致的未来帧`

简单来说，**MultiWorld** 就像一个 **“拥有上帝视角和精准指挥能力的导演”**：它先看清整个3D片场（全局状态），然后精确指挥每个演员（智能体）该做什么（动作条件），最后一次性拍出所有机位（视角）都连贯无误的镜头（未来视频）。
